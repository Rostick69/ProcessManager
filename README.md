# Диспетчер процессов — Лабораторная работа №2

**WPF-приложение для управления процессами Windows**  
с изменением приоритета, привязкой к ядрам и деревом иерархии 🚀

## Что умеет приложение

Это полноценный диспетчер задач в стиле лабы — но с расширенными возможностями, которых нет в стандартном Диспетчере Windows:

- Список всех процессов с **автообновлением каждые 4 секунды**  
- Подробная информация: PID, имя, приоритет, память (МБ), количество потоков, время CPU  
- **Изменение приоритета** — от Idle до RealTime (с предупреждением, чтобы не сломать систему)  
- **CPU Affinity** — чекбоксы для каждого ядра, маска в hex и binary, изменения сохраняются даже после обновления  
- Таблица **потоков** процесса — ID, приоритет, состояние (Wait, Running и т.д.), время CPU  
- **Дерево процессов** слева — иерархия родитель-потомок (explorer запускает chrome, RuntimeBroker и т.д.)  
- Выбор в дереве или таблице → сразу обновляется вся правая панель  

Всё работает стабильно, выделение не слетает, чекбоксы ядер остаются на месте после автообновления.

## Как запустить проект

1. Откройте **Visual Studio** (Community 2022 или новее)  
2. Установите .NET Framework 4.8 (если не установлен)  
3. Откройте файл решения: `ProcessManagerLab.sln`  
4. Нажмите **F5**

Требования:  
- Windows 10 / 11  
- .NET Framework 4.8

## Почему это круто

- Дерево процессов строится рекурсивно и обновляется вместе со списком  
- Чекбоксы ядер создаются динамически (по количеству ядер в системе)  
- Состояние сохраняется даже после автообновления (PID + маска)  
- Исключения обрабатываются — приложение не падает на системных процессах
